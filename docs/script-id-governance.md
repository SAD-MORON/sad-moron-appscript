# Script ID Governance

## Script IDs Are References, Not Secrets
Script IDs are used as references to Apps Script projects. They must not be treated as secrets or credentials.

## Script Ownership Requirements
Each script must have a clearly defined owner. Ownership must be documented and traceable.

## Deployment Traceability
All deployments must be traceable to a specific script ID and package. No undocumented deployments are permitted.

## Mapping Between Package and Script ID
A clear mapping between each package and its associated script ID must be maintained and documented.

## No Undocumented Script Execution
No script may be executed without documentation and governance approval.
