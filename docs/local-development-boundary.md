# Local Development Boundary for Apps Script

- All development must occur within the local repository unless explicitly approved.
- No credentials, tokens, or sensitive files are to be committed.
- .clasp.json, credentials, and environment files must be excluded from Git.
- Local deployment is allowed for testing, but no production deployment without governance review.
- Script IDs are references only; do not use for unauthorized deployment.
- Maintain strict separation between local code and deployed Apps Script.
- VSCode and CLASP workflows must follow SAD-MORON-FRAMEWORK governance.
